二叉树基础

完全二叉树

在完全二叉树中，除了最后一层外，其他层的节点都是满的，并且最后一层的节点都尽可能地靠左排列。简单来说，完全二叉树是一棵高度为h的二叉树，除了最后一层外，其他层的节点都是满的，最后一层的节点从左到右连续排列。

如何判断

当给出一张二叉树的图时，你可以按照以下步骤来判断它是否是完全二叉树：

  1. 从根节点开始，按照从上到下、从左到右的顺序遍历二叉树的节点。

  2. 在遍历过程中，观察每个节点的子节点情况：

    • 如果一个节点没有左子节点但有右子节点，那么它不是完全二叉树。
    • 如果一个节点有左子节点但没有右子节点，并且后面还有任何节点，那么它不是完全二叉树。

  3. 如果遍历过程中没有违反上述两个条件，且遍历结束后所有节点都被访问到，则该二叉树是完全二叉树。

满二叉树

在一个满二叉树中，所有的节点都具有0个或2个子节点，不存在只有一个子节点的节点。每一层的节点数量都达到了最大值。

深度和高度

  • 深度：一个节点的深度是从根节点到该节点的唯一路径上的边数。根节点的深度为0。
  • 高度：一个节点的高度是从该节点到最深叶子节点的最长路径上的边数。叶子节点的高度为0。

对于整棵树：

  • 树的深度：通常指的是根节点的深度，总是为0。
  • 树的高度：指的是根节点的高度，也就是从根节点到最深叶子节点的最长路径上的边数。

定义一个节点类

每个节点通常包含以下成员：

  1. 节点的值（例如，一个整数）。
  2. 指向左子节点的指针。
  3. 指向右子节点的指针。

插入节点

  1. 从哪里开始插入：通常，我们从根节点开始，然后按照层次顺序（从上到下，从左到右）查找可以插入新节点的位置。

  2. 确定哪个节点需要子节点：我们需要找到第一个没有左子节点或右子节点的节点。这意味着我们需要按照层次顺序遍历树，直到找到这样的节点。

  3. 使用的数据结构：为了按层次顺序遍历树，我们可以使用队列。开始时，我们将根节点放入队列。然后，每次从队列中取出一个节点，检查它是否有左子节点和右子节点。如果没有，则在该位置插入新节点。如果有，则将其子节点加入队列，以便稍后检查。

现在，让我们尝试将这个过程转化为步骤：

  1. 创建一个队列，并将根节点加入队列。
  2. 当队列不为空时，执行以下操作：
    a. 从队列前端取出一个节点。
    b. 检查该节点的左子节点。如果没有左子节点，将新节点作为左子节点插入并退出。否则，将左子节点加入队列。
    c. 检查该节点的右子节点。如果没有右子节点，将新节点作为右子节点插入并退出。否则，将右子节点加入队列。

代码分析

  1. 为什么void insert(TreeNode*& root, int value)中要用TreeNode*& root，而不是TreeNode* root呢？
    很好的问题！

在函数 void insert(TreeNode*& root, int value) 中，参数 TreeNode*& root 是一个对指针的引用。这意味着我们不仅可以修改 root 指向的内容（即树的节点），还可以修改 root 指针本身的值。

考虑以下情况：如果树是空的，我们需要创建一个新的根节点。如果我们只传递一个普通的指针（TreeNode* root），我们只能修改指针指向的内容，但不能修改指针本身的值。这意味着，即使我们在函数内部为 root 分配了新的内存，调用函数的外部代码仍然会看到一个空指针。

通过传递一个对指针的引用（TreeNode*& root），我们可以确保在函数内部对 root 进行的任何修改都会反映到调用函数的外部代码中。这样，当我们在函数内部为 root 分配新的内存时，调用函数的外部代码也会看到这个新的地址。

简而言之，使用 TreeNode*& root 允许我们在函数内部修改指针的值，这在处理空树或需要修改根节点的情况下是必要的。
