顶层const和底层const

  • **底层 const**：影响指针所指向的对象。
  • **顶层 const**：影响指针本身。

底层const

const int* p; 或者 int const* p;：这两种都表示 p 是一个指针，它指向一个 int 类型的常量。这里的 const 是底层 const，因为它影响的是指针所指向的对象。

顶层const

当你想要指针本身也是一个常量时，你应该写 int* const p;。这里的 const 是顶层 const，因为它影响的是指针本身，而不是指针所指向的对象。

如果你想要指针本身也是常量，且指向一个 int 类型的常量，你应该这样写：const int* const p; 或者 int const* const p;。

结构体和类

在 C++ 中，结构体 (struct) 和类 (class) 都是用户定义的数据类型，允许你创建包含数据成员和成员函数的复合类型。它们之间的主要区别在于默认的访问权限：

  1. **结构体 (struct)**：默认的成员访问权限是 public。
  2. **类 (class)**：默认的成员访问权限是 private。

何时使用结构体，何时使用类？

  1. 简单的数据聚合：当你仅需要一个简单的数据聚合，没有太多的行为（即成员函数）时，结构体可能更为合适。例如，表示一个二维点、颜色、范围等。

  2. OOP 设计：当你的设计遵循面向对象编程 (OOP) 原则时，例如封装、继承、多态等，使用类可能更为合适。

  3. 访问权限：如果你希望成员默认为 private，使用类。如果你希望它们默认为 public，使用结构体。

  4. 风格和约定：在某些编程文化或代码库中，可能有关于何时使用类和结构体的约定。遵循团队或项目的约定总是一个好主意。

最终，选择结构体还是类更多地取决于你的设计意图和个人或团队的编码风格。在 C++ 中，由于结构体和类的功能几乎相同，因此二者可以互换使用，只要它们在语义上是合理的。

*p++; 这里的运算顺序是什么？

在表达式 *p++ 中，运算的顺序取决于运算符的优先级和结合性。在这个例子中，我们有两个运算符：后置 ++（递增）运算符和 *（解引用）运算符。

后置 ++ 运算符的优先级高于 * 运算符，因此它会先执行。

但是，尽管后置 ++ 的优先级更高，它的效果（即增加 p 的值）是在整个表达式计算完之后才发生的。因此，*p++ 的效果是先解引用 p，然后再递增 p。

数组形参

你提到的确实是一个很重要的话题。在 C++ 中，当我们在函数中使用数组作为参数时，实际上是传递数组的指针，而不是整个数组。这意味着：

  1. 函数内部无法确定数组的大小，除非我们以其他方式传递这个信息。
  2. 由于传递的是数组的指针，函数内部对数组的修改会影响到原始数组。

Vector更好

在实际的 C++ 开发中，std::vector 是非常受欢迎的选择，尤其当你不确定数据的数量或需要灵活性时。使用 std::vector 有以下几个好处：

  1. 动态大小：与数组不同，std::vector 可以在运行时动态地改变大小。
  2. 容易传递：你可以轻松地将整个 std::vector 或其引用传递给函数。
  3. 内置功能：std::vector 提供了许多有用的内置函数，如 push_back(), size(), begin(), end() 等。
  4. 安全性：与裸数组相比，使用 std::vector 更容易避免诸如越界访问之类的错误。
